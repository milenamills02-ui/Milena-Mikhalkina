# Прогнозирование кредитного дефолта

Курсовой проект по машинному обучению. Задача - построить модель бинарной классификации для предсказания Credit Default (1 - дефолт, 0 - нет дефолта). Метрика качества - F1-score для класса 1, должна быть > 0.5.

## Файлы в проекте

- credit_default_solution.ipynb - ноутбук с решением
- course_project_train.csv - обучающий датасет
- course_project_test.csv - тестовый датасет
- predictions_only_target.csv - итоговый файл с прогнозами (только колонка Credit Default)
- predictions.csv - прогнозы с id (на случай если нужен id)
- README.md - описание проекта

## Как запустить

1) Установить зависимости:
pip install pandas numpy scikit-learn matplotlib seaborn

2) Положить course_project_train.csv и course_project_test.csv в ту же папку, где лежит ноутбук.

3) Открыть credit_default_solution.ipynb в Jupyter / JupyterLab / VS Code и запустить все ячейки по порядку.

После выполнения появятся файлы predictions_only_target.csv и predictions.csv.

## Что сделано

Сначала проверила данные: распределение классов, пропуски и выбросы. Классы несбалансированы (примерно 72/28), поэтому в моделях использовалась балансировка через class_weight.

В Current Loan Amount встречалось значение 99999999. Это не реальная сумма, поэтому заменено на пропуск.

Пропуски в числовых признаках заполнялись медианой по train. Пропуски в категориальных - самым частым значением. Категории закодированы, чтобы модель могла их использовать. Часть преобразований объединена в Pipeline, чтобы одинаково применять их при обучении и кросс-валидации.

Добавила несколько простых признаков (например, отношение долга к доходу и использование кредитного лимита), чтобы улучшить качество.

Дальше обучила базовую модель (логистическая регрессия) и сравнила с другими моделями. Лучший результат на валидации показал Random Forest. Для повышения F1 подбирался порог классификации, а не использовался фиксированный 0.5.

Дополнительно сделана своя реализация логистической регрессии на основе градиентного спуска и проведено сравнение со sklearn-версией.

## Результат

Лучшая модель дает F1-score для класса 1 больше 0.5 на валидации. Финальная модель обучена на всем train и использована для прогнозов на test. Прогнозы сохранены для всех строк test в исходном порядке.
